# Library Management / 漂流书项目记录

## 当前结论

- 第一阶段真实需求已收敛为“`一本书的漂流`”互动网页，而不是完整图书借阅系统。
- 核心目标是：`100` 个二维码对应 `100` 个独立书页，每页展示一本书，并支持最多 `10` 层的单链接龙留言。
- 学生端以手机响应式 Web 为主，后台提供页面维护、留言审核、二维码管理和页面重置。

## 已确认的产品边界

- 每个二维码页面代表一本具体的书，不是泛活动页。
- 留言结构采用“单链式接龙”，不是树状论坛。
- 每页最多 `10` 层，达到上限后页面锁定为只读。
- 留言采用“先审核后公开”。
- 页面满 `10` 层后，由管理员手动重置开启新一轮。
- 首版不做完整借阅管理、不做个性化推荐、不做统一身份认证。

## GitHub 调研结论

- GitHub 上没有找到能直接满足“二维码书页 + 10 层接龙 + 审核后台”的现成成品。
- 最接近、最适合作为改造底座的是 `NEET64/book-world`：
  - 仓库地址: <https://github.com/NEET64/book-world>
  - 技术栈: MERN (`MongoDB` + `Express` + `React` + `Node.js`)
  - 已有能力: 书籍详情页、评分、评论、嵌套回复、收藏、管理员管理书籍和用户
  - 已确认许可证: `MIT`
- 该项目适合复用的部分：
  - 书籍详情展示
  - 用户与管理员权限
  - 评论/评分基础框架
  - 后台管理基本结构
- 该项目不满足当前需求的部分：
  - 100 个独立二维码页
  - 单链 10 层接龙
  - 留言审核流
  - 页面满层锁定与轮次重置
  - 校园品牌元素和活动模板化管理

- 缺留言审核状态
- 缺单链 10 层控制
- 缺页面轮次与重置机制
- 缺批量初始化 100 页及二维码映射

## 目标改造方向

保留 `book-world` 现有的书籍展示、认证和后台骨架，不直接沿用其 `Review + Comment` 论坛树结构来实现漂流书。

更稳的方向是新增两类核心实体：

- `DriftPage`
  - 绑定书籍
  - 二维码码值/访问路径
  - 当前轮次
  - 当前层级
  - 页面状态
  - 校园展示素材

- `DriftMessage`
  - 所属页面
  - 轮次
  - 层级 `1-10`
  - 学生 ID
  - 留言内容
  - 审核状态
  - 审核记录
